# idea-pipeline shortlist template (stage 4)

`_vision/strategy/_pipeline/top-5.md` is the artifact the CEO takes into the user-meeting. One page per idea, strict format — the user reads these cold.

## File skeleton

```markdown
# Top-5 Ideas for User Review — <YYYY-MM-DD>

> Generated by idea-pipeline. CEO's review notes follow each card.
> User picks 1–2 at the user-meeting.

---

## 1. <Idea name>

**Elevator** (≤ 30 words): <positioning-strategist's elevator>

**Wedge**: <one line>

**Who wants this**: <one-line audience>

**Numbers**
- TAM: <$X> · SAM: <$Y> · SOM 3y: <$Z> (confidence: L/M/H · basis: <source>)
- Effort: <weeks range> · Composite: <0-5>

**Proof**
- <cited fact 1>
- <cited fact 2>

**Risks (top 2)**
- <risk · mitigation or "open">
- <risk · mitigation or "open">

**ETA if greenlit**: <target launch date range>

**CEO's read**: <1-2 sentences — why this is on the list>

---

## 2. <Idea name>

(same skeleton)

---

## 3. <Idea name>

(same skeleton)

---

## 4. <Idea name>

(same skeleton)

---

## 5. <Idea name>

(same skeleton)

---

## Context

- Strategic theses this top-5 reinforces: <list>
- Active OKR(s) each idea traces to: <table>
- What we said no to this round: <3-5 bullets · one-line rejection>

## CEO's recommendation (optional)

<Rank and why — but user has final call. If CEO strongly prefers one, say so and cite the reason. Empty is also fine.>
```

## Caps

- 1 page per idea (≤ 40 lines).
- Total file ≤ 250 lines.
- No embedded images. Text only — this is read in chat.

## One-pager rules

- Every number cited. Missing citation = blank = material defect.
- No jargon that isn't in `_vision/brand-guide.md`.
- ETA is a range, never a single date.
- Risks always ≥ 2. No risk-free idea exists.

## Handoff

- File written → `user-meeting` skill invoked with `top_5_path: _vision/strategy/_pipeline/top-5.md`.
- No edits after handoff. If user requests changes, generate a new `top-5-<date>-v2.md` and ADR the pivot.
